หากท่านได้รับมอบหมายให้รับหน้าที่พัฒนาระบบ POS คิดเงินหน้าร้านเกี่ยวกับระบบ Fitness และมีการประเมินจาก Developer lead หรือ Project Manager ว่าต้องใช้เวลาพัฒนาทังหมด 120 วันและงานในส่วนที่ต้อง Develop กว่า 50 Function ย่อยใน Project นี้ที่ท่านรับผิดขอบทางทีมไม่ได้กําหนดเวลาเข้าออกงานหรือต้องเข้ามาทํางานที่บริษัทฯแต่เน้นผลลัพธ์ของงานเป็นสําคัญท่านจะออกแบบวิธีการทํางานของท่านอย่างไรจะใช้วิธีรายงานความคืบหน้างานอย่างไรทําอย่างไรเมื่อติดปัญหาและจะจัดการปัญหาอย่างไรหากเล็งเห็นว่างาเริ่มช้ากว่าปกติ

POS 120 Days 

- ออกแบบวิธีการทํางานของท่านอย่างไร

    เมื่อได้รับงานจากหัวหน้าแล้วทำการวางแผนงานที่ได้รับโดยการจัดระดับความสำคัญของ Project ย่อยที่ได้รับมานั้นว่างานไหนที่ต้องส่งต่อให้กับทีมทำต่อไหม หรือ สิ่งไหนสามารถทำคนเดียวได้เลย 

    ถ้างานที่ได้รับมาต้องส่งต่อให้ทีมก็จะต้องนำมาจัดลำดับก่อนเพื่อที่งานจะไปส่งต่อไปได้โดยไม่ต้องเสียเวลารองานจากเรา
    หลังจากจัดลำดับแล้วการทำการจัดตารางงานเวลาว่าต้องทำงานนานแค่ไหน และ พัก นานเท่าไร ควรกำหนดเป้าหมายในการทำงาน กำหนดเป้าหมายในแต่ละวันว่าต้องทำส่วนไหนให้เสร็จโดนแบ่งเป็น 

    ระยะ สั้น เป็นกสรกำหนดเป้าหมายในแต่วัน
    ระยะ กลาง เป็นการกำหนดเป้าในแต่ละสัปดาห์
    ระยะ ยาว เป็นการกำหนดเป้าระยะเวลาการทำงานทั้ง Project


- วิธีรายงานความคืบหน้างานอย่างไรทําอย่างไรเมื่อติดปัญหา

    ส่วนการวัดผลการทำงานและรายงานความคืบหน้าให้กับ Project Manager นั้น จะดำเนินการรายงาน โดย ทุกวันจะรายงานผ่านช่องทาง Chat Line ทุกๆ สัปดาห์จะมีการนัดประชุมเผื่ออัพเดตการทำงานและปัญหาที่เกิดขึ้น เพื่อที่จะได้หาทางแก้ไขได้ถูกต้อง


- จัดการปัญหาอย่างไรหากเล็งเห็นว่างานเริ่มช้ากว่าปกติ

    การจัดการงานเริ่มล่าช้าอาจจะต้องสอบถามความช่วยจากทีมว่าสามารถนำส่วนไหนที่เราไม่ถนัดนำไปช่วยทำ เพื่อที่จะได้โฟกัสการทำงานในสิ่งที่เราทำงานได้เร็วที่สุด หลังจากเสร็จแล้ว จึงค่อยนำส่วนที่ทีมทำค้างไว้มาดำเนินการต่อให้เสร็จทันระยะเวลาที่กำหนด
